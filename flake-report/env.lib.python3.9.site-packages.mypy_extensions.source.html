76 TypedDict.__doc__ = """A simple typed name space. At runtime it is equivalent to a plain dict.
77  
78 TypedDict creates a dictionary type that expects all of its
79 instances to have a certain set of keys, with each key
80 associated with a value of a consistent type. This expectation
81 is not checked at runtime but is only enforced by typecheckers.
82 Usage::
83  
84 Point2D = TypedDict('Point2D', {'x': int, 'y': int, 'label': str})
85 a: Point2D = {'x': 1, 'y': 2, 'label': 'good'} # OK
86 b: Point2D = {'z': 3, 'label': 'bad'} # Fails type check
87 assert Point2D(x=1, y=2, label='first') == dict(x=1, y=2, label='first')
88  
89 The type info could be accessed via Point2D.__annotations__. TypedDict
90 supports two additional equivalent forms::
91  
92 Point2D = TypedDict('Point2D', x=int, y=int, label=str)
93  
94 class Point2D(TypedDict):
95 x: int
96 y: int
97 label: str
98  
99 The latter syntax is only supported in Python 3.6+, while two other
100 syntax forms work for Python 2.7 and 3.2+
101 """
